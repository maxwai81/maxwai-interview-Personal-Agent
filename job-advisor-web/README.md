# Job Advisor Web

A web form and interactive dashboard for job seekers to capture application details and view Job Advisor fit analysis results.

## Features

- **Form**: Company name, job title, job URL, resume upload, LinkedIn URL
- **Payload generation**: JSON output for Cursor's Job-Advisor-for-Job-Seekers subagent
- **Dashboard (index.html)**: Parse and visualize Job Advisor reports (overall score, granular scores, strengths, gaps, recommendations)
- **Interactive Dashboard (job-fit-dashboard.html)**: Rich visualization with radar/bar charts, job requirements table, company highlights, and tabbed recommendations — loads from `job-fit-dashboard.json` generated by the agent

### Cursor / Cloud agents (automation)

- **Job-Advisor-for-Job-Seekers** should run **automatically** for job-fit / application-analysis requests (see `.cursorrules` and `AGENT_INSTRUCTIONS.md`).
- After each run, agents should **merge to `main`** (or open a non-draft PR with auto-merge) per Job-Advisor **Step 6**, so GitHub Pages and `main` stay up to date without an extra user prompt.

## Deploy to GitHub Pages

1. Push this repo to GitHub
2. Go to **Settings → Pages**
3. Under **Source**, select **Deploy from a branch**
4. Choose branch (e.g. `main`) and folder **/ (root)**
5. Save — your site will be at `https://<username>.github.io/<repo>/job-advisor-web/`

Open the `job-advisor-web` path to use the app.

## Local workflow (automated)

1. **Start the server** (from project root):
   ```bash
   npm run job-advisor
   ```
   Or: `node job-advisor-web/server.js`  
   If port 8765 is in use: `PORT=8766 npm run job-advisor`

2. **Open** http://localhost:8765

3. **Fill the form** → click **Save request & run in Cursor**

4. **In Cursor**, say: *"Process job-advisor-web/request.json"* or *"Run Job Advisor for the request in request.json"*

5. **Refresh the Dashboard** tab or click **Refresh from file** — the report auto-loads from `report.md`

No manual copy/paste needed.

## Workflow (with server)

1. **Fill the form** with company (required), job title, job URL, LinkedIn, and optionally upload a resume
2. **Save request** — click the button; request is written to `request.json`
3. **In Cursor**, say: *"Process job-advisor-web/request.json"*
4. **Agent** runs company research + resume matching and writes the report to `report.md`
5. **Dashboard** auto-loads from `report.md` or click **Refresh from file** — no manual paste needed

Without the server (static mode): form shows JSON to copy; paste report manually into the Dashboard.

**Alternative**: The agent now generates `job-fit-dashboard.json`. Open `job-fit-dashboard.html` in a browser (or click "Load job-fit-dashboard.json") to view the interactive dashboard with charts, job requirements table, company highlights, and tabbed recommendations.

## Direct invocation (no form)

In Cursor, invoke the Job Advisor directly:

```
Run job fit for [Company], [Job Title]. My LinkedIn: [url]
```

The agent will run company research + resume matching, write `report.md` to `job-advisor-web/`, and give you a link to the dashboard. Open http://localhost:8765 (if the server is running) or `job-advisor-web/index.html` to view results.

## Per-run dashboards (`runs/<slug>/`)

Each Job Advisor run can also produce a **standalone** dashboard that matches the `index.html` Dashboard tab (same CSS/JS):

- `runs/<slug>/dashboard.html` — loads `./job-fit.json` (same schema as `job-fit-dashboard.json`); includes an embedded copy for `file://` viewing (see `runs/README.md`)
- Shared styles: `dashboard-shared.css` · logic: `dashboard-run-app.js`
- Template to copy: `run-dashboard-template.html` · example: `runs/2026-03-19-moveworks-sse/`

See `runs/README.md` for slug naming. After a run, the agent should `git add job-advisor-web/`, commit, and push when possible so GitHub Pages serves the new files.

## Files

| File | Purpose |
|------|---------|
| `index.html` | Form + markdown-based dashboard (single-page app) |
| `dashboard-shared.css` | Shared ServiceNow-style dashboard styles (index + per-run pages) |
| `dashboard-run-app.js` | Renders charts/tabs from `job-fit.json` for per-run pages |
| `run-dashboard-template.html` | Copy into `runs/<slug>/dashboard.html` for new runs |
| `job-fit-dashboard.html` | Interactive dashboard with charts, loads from JSON |
| `job-fit-dashboard.json` | Structured data for the dashboard (generated by agent) |
| `runs/` | Per-run `dashboard.html` + `job-fit.json` per analysis |
| `schema.json` | JSON schema for form payload |
| `sample-report.md` | Sample report for testing the markdown dashboard |

## Sample report

To test the markdown dashboard without running the subagent, copy the contents of `sample-report.md` and paste into "Load Results". To test the interactive dashboard, `job-fit-dashboard.json` is pre-populated with a ServiceNow–Max example.
